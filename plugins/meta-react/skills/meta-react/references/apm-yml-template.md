# apm.yml Template & Customization

The shipped template at `templates/apm.yml` is intentionally minimal. This file documents the schema fields the agent may set or merge, and how to extend the manifest for project-specific needs.

## Shipped template

```yaml
name: <project-name>
version: 1.0.0
description: React project scaffolded with meta-react
targets:
  - claude
  - copilot
  - agent-skills
dependencies:
  apm:
    - vercel-labs/agent-skills/skills/react-best-practices
  mcp: []
```

> Note: the field is `targets:` (plural), block-style list. Current APM CLI versions reject the singular `target:` and the inline-array form `[a, b]` — keep block style.

## Fields the agent MAY set

| Field | When | How |
|-------|------|-----|
| `name` | Always | Substitute `<project-name>` from `package.json` `name` or directory basename |
| `targets` | Always | Block-style list of detected targets (`claude`, `copilot`, `agent-skills`). Always include `agent-skills`. |
| `dependencies.apm` | On merge | Append missing deps; never overwrite user pins |
| `dependencies.mcp` | On merge | Append missing servers; never overwrite |

## Fields the agent MUST NOT overwrite

| Field | Reason |
|-------|--------|
| `version` | User project lifecycle |
| `description` | User authorship |
| `author`, `license` | User attribution |
| `scripts` | User automation |
| Any user-added top-level key | Forward-compat: APM reserves unknown keys |

## Adding APM dependencies

The `dependencies.apm` list takes shorthand or URL forms (see APM manifest spec). Useful additions for React projects:

```yaml
dependencies:
  apm:
    - vercel-labs/agent-skills/skills/react-best-practices
    - vercel-labs/agent-skills/skills/typescript-types
    - microsoft/apm-sample-package#v1.0.0   # pinned tag
```

Each entry can also be an object with extra fields (skills filter, ref, etc.):

```yaml
dependencies:
  apm:
    - source: vercel-labs/agent-skills
      ref: main
      skills:
        - react-best-practices
        - no-use-effect
```

## Adding MCP servers

```yaml
dependencies:
  mcp:
    - name: github
      transport: http
      url: https://api.githubcopilot.com/mcp
    - name: filesystem
      transport: stdio
      command: npx
      args: ["-y", "@modelcontextprotocol/server-filesystem", "${workspaceFolder}"]
```

Env-var placeholders (`${env:TOKEN}`) are translated to each runtime's native syntax at install time — see the APM CLI docs.

## Target selection

| Target | Effect |
|--------|--------|
| `claude` | Emits `CLAUDE.md` at root, deploys to `.claude/` |
| `copilot` | Emits `AGENTS.md` at root, deploys to `.github/` |
| `agent-skills` | Cross-client `.agents/skills/` (use alongside others) |
| `cursor`, `codex`, `gemini`, `windsurf`, `opencode` | Provider-specific |

Use a block-style list when targeting multiple clients:

```yaml
targets:
  - claude
  - copilot
  - agent-skills
```

Valid target values (per `apm targets` output): `agent-skills`, `claude`, `codex`, `copilot`, `cursor`, `gemini`, `opencode`, `windsurf`.

## Merge example

User's existing `apm.yml`:

```yaml
name: my-app
version: 0.3.1
description: Internal tools dashboard
dependencies:
  apm:
    - vercel-labs/agent-skills/skills/react-best-practices#main
```

After running meta-react with both targets detected, the merged file:

```yaml
name: my-app                # preserved
version: 0.3.1              # preserved
description: Internal tools dashboard  # preserved
targets:                    # added (block-style list, plural)
  - claude
  - copilot
  - agent-skills
dependencies:
  apm:
    - vercel-labs/agent-skills/skills/react-best-practices#main  # preserved with user pin
  mcp: []                   # added (was missing)
```

The user's `#main` ref is kept because the **canonical key** (`vercel-labs/agent-skills/skills/react-best-practices`, ignoring `#ref`) matches the template entry — see `install-workflow.md` "Canonical key for dedup" for the full algorithm.

## Lockfile

`apm.lock.yaml` is generated by `apm install` and SHOULD be committed. The meta-react skill never edits it directly.
